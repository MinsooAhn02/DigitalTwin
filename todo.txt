# Traffic Digital Twin — 현황 정리 (2026-04-28)

==========================================================
## 처음 구조 vs 지금
==========================================================

[초기]
- REPLAY_MODE=true : JSON 파일(real_world_track_data.json)에서 차량 데이터를 재생
- 실제 CCTV 영상 없음, 지도에만 차량 점이 움직이는 수준
- YOLO 탐지 없음

[지금]
- REPLAY_MODE=false : ITS 국가교통정보센터 API에서 실제 CCTV HLS 스트림 수신
- 지도에서 CCTV 아이콘 클릭 → 실시간 영상 팝업 (HLS.js 플레이어)
- YOLO 탐지 탭에서 바운딩박스 포함 실시간 영상 확인 가능
- 탐지된 차량 데이터가 지도에 표시 + 사이드바 통계(속도, LOS, 차종, 카운터) 업데이트

==========================================================
## 구현 완료 목록
==========================================================

- ITS OpenAPI 연동 : 현재 뷰포트 기준 CCTV 목록 조회 (/cctvs)
- HLS 영상 플레이어 : hls.js 기반, 스트림 오류 자동 복구
- CCTV 패널 : 560px 플로팅 패널, 드래그로 위치 이동 가능
- 탭 전환 : "실시간" / "YOLO 탐지" 탭
- YOLO 탐지 파이프라인
    브라우저 캔버스에서 프레임 캡처
    → WebSocket (ws://localhost:8000/ws/detect)으로 전송
    → 백엔드 YOLOv8 추론
    → 어노테이션 이미지 반환 + 지도/사이드바 브로드캐스트
- ByteTrack 다중 차량 추적 (supervision 0.27 호환)
- 속도 / LOS / 과속·꼬리물기·병목 경보
- 카메라 전환 : CCTV 클릭 시 /switch-camera 호출, 트래커 리셋

==========================================================
## 주요 변경 이유
==========================================================

[YOLOv8x → YOLOv8n, imgsz 640 → 320]
- ITS HLS URL에는 만료 토큰이 붙어있어 OpenCV 백엔드가 직접 열지 못함
- 대신 브라우저가 이미 재생 중인 영상에서 캔버스로 프레임을 캡처해 WebSocket 전송
- 이 구조는 CPU에서 추론하므로 속도 확보를 위해 경량 모델(n)과 저해상도(320) 선택
- yolov8x CPU : ~500ms/프레임(2fps), yolov8n CPU : ~50ms/프레임(20fps)

[supervision ByteTracker → ByteTrack]
- supervision 0.27에서 API 이름 변경 (track_buffer → lost_track_buffer)
- 수정 전에는 서버 시작 시 조용히 크래시 → 차량 추적 전혀 안 됨

## 남은 개선 과제
==========================================================

[GPU 가속 — 완료]
- torch 2.6.0+cu124 설치 완료, CUDA available: True
- 효과 : 추론 속도 5~10배 향상 예상
- 다음 단계 : yolov8n → yolov8s 또는 yolov8m 으로 업그레이드 가능
             imgsz 320 → 640 복원으로 탐지 정확도도 함께 개선

[속도 계산 정확도]
- 현재는 고정 FPS=30 기준으로 속도 계산 → 실제 탐지 주기와 불일치
- 개선안 : 프레임별 실제 timestamp_ms 델타 사용
