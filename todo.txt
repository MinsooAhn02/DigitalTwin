C. 논문 아직 안 쓴 설명들
──────────────────────────────────────────────────────

[ ] ByteTrack 대비 OC-SORT / BotSort 나은 점 설명
      → OC-SORT: occlusion 중에도 ID 유지 (observation-centric re-ID)
      → BotSort: OSNet appearance ReID로 외형 기반 재매칭 → ID switch 감소
      → ByteTrack: 빠르지만 ReID 없어서 occlusion 후 ID switch 많음
      위치: Related Work §2 또는 Implementation §4.1

[ ] YOLOv8 → YOLO26 교체 이유 설명
      → NMS-free end-to-end 구조 → per-frame bbox 일관성 높음
      → ground-contact point jitter 감소 → speed estimate 안정
      → TensorRT graph 단순 (NMS layer 없음) → latency 낮고 variance 작음
      → YOLOv8은 legacy fallback으로 유지
      위치: Related Work §2 (이미 일부 있음, 보강 필요)

[ ] 각 로직 설명 보강

      → curved transform 2단계 (Algorithm 1) — 이미 있음
      → auto-calibration pose solver — 이미 있음
      → vehicle_calib.json (B, C 모델) — 이번 세션에 추가함 ✓
